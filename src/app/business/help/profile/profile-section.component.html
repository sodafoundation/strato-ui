A profile is a set of configurations on service capabilities (including resource tuning, QoS) of storage resources. A profile must be specified when volume is created.

To create a new profile click on the Create button in the Profile page and fill the details.

  • Name: has to be unique
  • Description: What is this profile for?
  • Storage Type: Block or File?
  • Provisioning Type: iSCSI, FC, RBD in case of Block type. NFS, CIFS in case of File type.
  • Storage Access Capability (only File type): read, write, execute.
    Additionally you can also set some policies like:
  • QoS Policy (Both Block and File type): Maximum IOPS, Minimum IOPS, Maximum BWS, Minimum BWS and Latency
  • Snapshot Policy (Both Block and File type): Enable Automatic snapshot, Schedule, Execution time and Retention.
  • Replication Policy (Block type only): Type, Mode, Period, Bandwidth, RGO, RTO, RPO, Consistency.
